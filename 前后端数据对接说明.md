## 数据格式说明

### 分页响应格式

当使用分页时，后端返回：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "results": [...],      // 当前页数据
    "count": 100,          // 总记录数
    "page": 1,             // 当前页码
    "page_size": 20,       // 每页数量
    "total_pages": 5,      // 总页数
    "next": "...",         // 下一页链接
    "previous": "..."      // 上一页链接
  }
}
```

### 非分页响应格式

不使用分页时，后端返回：
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": [...]  // 直接是数组
}
```

### 前端处理方式

前端统一使用以下方式处理：
```javascript
const response = await api.market.getPrices(params)
const resData = response.data?.data || response.data || {}

// 判断是否有分页
if (resData.results) {
  // 分页格式
  tableData.value = resData.results
  total.value = resData.count
} else if (Array.isArray(resData)) {
  // 数组格式
  tableData.value = resData
  total.value = resData.length
}
```

## 常见问题

### Q1: 前端显示"加载数据失败"

**原因**: 
- 后端服务器未启动
- API路径错误
- CORS配置问题

**解决**:
1. 确认后端运行在 `http://localhost:8000`
2. 检查浏览器控制台（F12）的错误信息
3. 确认 `frontend/vue.config.js` 中的代理配置正确

### Q2: 数据格式不匹配

**原因**: 
- 后端返回格式与前端期望不一致

**解决**:
- 检查浏览器 Network 标签，查看实际返回的数据格式
- 根据实际格式调整前端代码

### Q3: 分页不工作

**原因**: 
- 后端未启用分页
- 前端分页参数传递错误

**解决**:
- 检查后端是否使用了 `StandardResultsSetPagination`
- 确认前端传递了 `page` 和 `page_size` 参数
