# 前端启动指南

## ⚠️ 重要提示

**前端和后端是分开运行的！** 需要同时运行两个服务器：
- **后端（Django）**: `http://localhost:8000`
- **前端（Vue.js）**: `http://localhost:8080`

## 启动步骤

### 1. 启动后端（Django）

在项目根目录执行：
```bash
# 激活虚拟环境（如果使用）
.venv\Scripts\activate

# 启动Django服务器
python manage.py runserver
```

后端会在 `http://localhost:8000` 运行

### 2. 启动前端（Vue.js）

**打开新的终端窗口**，进入前端目录：

```bash
# 进入前端目录
cd frontend

# 检查是否已安装依赖
# 如果 node_modules 目录不存在，需要先安装依赖
npm install

# 启动前端开发服务器
npm run serve
```

前端会在 `http://localhost:8080` 运行

### 3. 访问应用

在浏览器中访问：**http://localhost:8080**

**不要访问 http://localhost:8000**（那是后端API，不是前端界面）

## 常见问题解决

### 问题1：界面没有变化

**解决方案：**

1. **确认前端服务器正在运行**
   - 检查终端是否有 "App running at: http://localhost:8080" 的提示
   - 如果没有，说明前端服务器没有启动

2. **清除浏览器缓存**
   - 按 `Ctrl + Shift + R` (Windows) 或 `Cmd + Shift + R` (Mac) 强制刷新
   - 或者按 `F12` 打开开发者工具，右键刷新按钮，选择"清空缓存并硬性重新加载"

3. **重启前端服务器**
   ```bash
   # 在 frontend 目录下
   # 按 Ctrl+C 停止服务器
   # 然后重新启动
   npm run serve
   ```

4. **检查是否有编译错误**
   - 查看终端是否有红色错误信息
   - 如果有错误，需要先修复错误

### 问题2：npm install 失败

**解决方案：**

```bash
# 使用国内镜像
npm install --registry=https://registry.npmmirror.com

# 或者使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 问题3：找不到 vue-cli-service

**解决方案：**

```bash
# 在 frontend 目录下
npm install --save-dev @vue/cli-service
```

### 问题4：端口 8080 被占用

**解决方案：**

修改 `frontend/vue.config.js`：
```javascript
module.exports = {
  devServer: {
    port: 8081,  // 改为其他端口
    // ...
  }
}
```

然后访问 `http://localhost:8081`

## 在 PyCharm 中运行

### 方法1：使用终端（推荐）

1. 在 PyCharm 底部打开 **Terminal** 标签
2. 启动后端：
   ```bash
   python manage.py runserver
   ```
3. 点击 Terminal 旁边的 **+** 号，打开新的终端标签
4. 在新终端中启动前端：
   ```bash
   cd frontend
   npm run serve
   ```

### 方法2：配置运行配置

1. 点击 PyCharm 右上角的运行配置下拉菜单
2. 选择 "Edit Configurations..."
3. 添加两个运行配置：
   - **Django Server**: 运行 `python manage.py runserver`
   - **Vue Dev Server**: 运行 `npm run serve` (在 frontend 目录)

## 验证步骤

1. ✅ 后端运行在 `http://localhost:8000`
   - 访问 `http://localhost:8000/admin/` 应该能看到 Django 管理后台

2. ✅ 前端运行在 `http://localhost:8080`
   - 访问 `http://localhost:8080` 应该能看到优化后的前端界面
   - 界面应该有：
     - 紫色渐变的导航栏
     - "农产品分析平台" Logo
     - 消息中心按钮
     - 现代化的卡片布局

3. ✅ 浏览器控制台没有错误
   - 按 `F12` 打开开发者工具
   - 查看 Console 标签，不应该有红色错误

## 快速检查清单

- [ ] 后端服务器已启动（`python manage.py runserver`）
- [ ] 前端服务器已启动（`npm run serve`）
- [ ] 访问的是 `http://localhost:8080`（不是 8000）
- [ ] 浏览器已清除缓存（Ctrl+Shift+R）
- [ ] 终端没有错误信息
- [ ] 浏览器控制台没有错误

## 如果还是不行

1. **检查文件是否保存**
   - 确保所有修改的文件都已保存（Ctrl+S）

2. **检查前端依赖是否完整**
   ```bash
   cd frontend
   npm list
   ```
   确保所有依赖都已安装

3. **重新安装依赖**
   ```bash
   cd frontend
   rm -rf node_modules  # Windows: rmdir /s node_modules
   npm install
   npm run serve
   ```

4. **查看终端输出**
   - 前端服务器启动时会显示编译信息
   - 如果有错误，会显示红色错误信息
   - 成功时会显示 "App running at: http://localhost:8080"
