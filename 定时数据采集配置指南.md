# 定时数据采集配置指南

本指南介绍如何配置系统每天自动采集网站数据。

## 一、方案选择

系统提供了三种定时任务方案，可根据需求选择：

### 方案一：APScheduler（推荐，简单易用）
- ✅ 无需额外服务（Redis/消息队列）
- ✅ 配置简单
- ✅ 适合中小型项目
- ❌ 单机部署

### 方案二：Celery + Celery Beat（生产环境推荐）
- ✅ 支持分布式
- ✅ 任务队列管理
- ✅ 适合大型项目
- ❌ 需要Redis服务
- ❌ 配置较复杂

### 方案三：系统Cron（最稳定）
- ✅ 系统级稳定性
- ✅ 不依赖Django进程
- ✅ 适合服务器环境
- ❌ 需要服务器权限

## 二、方案一：APScheduler（推荐）

### 2.1 安装依赖

```bash
pip install django-apscheduler apscheduler
```

### 2.2 配置Django

已在 `settings.py` 中配置好，无需额外配置。

### 2.3 数据库迁移

```bash
python manage.py migrate
```

这会创建定时任务相关的数据库表。

### 2.4 启动定时任务

定时任务会在Django启动时自动启动。启动Django服务器：

```bash
python manage.py runserver
```

### 2.5 默认定时任务

系统默认配置了以下定时任务：

- **每天凌晨2点**：执行数据采集
- **每天上午10点**：执行数据采集
- **每天下午6点**：执行数据采集

### 2.6 自定义定时任务

编辑 `apps/data_etl/scheduler.py`，修改或添加定时任务：

```python
# 每天凌晨2点执行
scheduler.add_job(
    daily_data_collection_task,
    trigger=CronTrigger(hour=2, minute=0),
    id='daily_data_collection',
    name='每日数据采集',
    replace_existing=True,
)

# 每小时执行一次
scheduler.add_job(
    daily_data_collection_task,
    trigger=CronTrigger(minute=0),  # 每小时的第0分钟
    id='hourly_data_collection',
    name='每小时数据采集',
    replace_existing=True,
)
```

### 2.7 查看定时任务状态

访问Django Admin：`http://localhost:8000/admin/`

在管理界面可以看到：
- **Django Job Executions**：定时任务执行记录
- **Django Jobs**：定时任务列表

### 2.8 手动执行定时任务

```bash
# 采集所有启用的数据源
python manage.py schedule_data_collection

# 采集指定数据源
python manage.py schedule_data_collection --source-id 1
```

## 三、方案二：Celery + Celery Beat

### 3.1 安装Redis

**Windows:**
```bash
# 下载Redis for Windows
# 或使用WSL安装
```

**Linux/macOS:**
```bash
sudo apt-get install redis-server  # Ubuntu/Debian
brew install redis  # macOS
```

启动Redis：
```bash
redis-server
```

### 3.2 配置Celery

创建 `agricultural_platform/celery.py`：

```python
import os
from celery import Celery

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'agricultural_platform.settings')

app = Celery('agricultural_platform')
app.config_from_object('django.conf:settings', namespace='CELERY')
app.autodiscover_tasks()
```

修改 `agricultural_platform/__init__.py`：

```python
from .celery import app as celery_app

__all__ = ('celery_app',)
```

在 `settings.py` 中添加：

```python
CELERY_BROKER_URL = 'redis://localhost:6379/0'
CELERY_RESULT_BACKEND = 'redis://localhost:6379/0'
CELERY_TIMEZONE = 'Asia/Shanghai'
CELERY_BEAT_SCHEDULE = {
    'daily-data-collection': {
        'task': 'apps.data_etl.tasks.daily_data_collection_task',
        'schedule': crontab(hour=2, minute=0),  # 每天凌晨2点
    },
}
```

### 3.3 启动Celery Worker

```bash
celery -A agricultural_platform worker -l info
```

### 3.4 启动Celery Beat

```bash
celery -A agricultural_platform beat -l info
```

## 四、方案三：系统Cron（Linux/macOS）

### 4.1 创建执行脚本

创建 `scripts/daily_collection.sh`：

```bash
#!/bin/bash
cd /path/to/Agricultural
source .venv/bin/activate
python manage.py schedule_data_collection >> logs/cron.log 2>&1
```

赋予执行权限：
```bash
chmod +x scripts/daily_collection.sh
```

### 4.2 配置Cron

编辑crontab：
```bash
crontab -e
```

添加定时任务：
```
# 每天凌晨2点执行数据采集
0 2 * * * /path/to/Agricultural/scripts/daily_collection.sh

# 每天上午10点和下午6点也执行
0 10 * * * /path/to/Agricultural/scripts/daily_collection.sh
0 18 * * * /path/to/Agricultural/scripts/daily_collection.sh
```

### 4.3 Windows任务计划程序

1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器：每天，时间：02:00
4. 操作：启动程序
   - 程序：`python`
   - 参数：`manage.py schedule_data_collection`
   - 起始于：项目目录路径

- `logs/agricultural.log`：应用日志
- `logs/cron.log`：Cron任务日志（如果使用Cron方案）

如果任务执行失败：
1. 查看日志文件了解错误原因
2. 检查数据源配置是否正确
3. 检查网络连接
4. 手动执行一次测试：
   ```bash
   python manage.py schedule_data_collection --source-id 1
   ```

## 六、最佳实践

### 6.1 采集时间建议

- **凌晨2-4点**：网站访问量低，数据更新完成
- **避开高峰期**：避免在网站访问高峰期采集
- **多次采集**：每天2-3次，确保数据及时性

### 6.2 数据源管理

- 在Django Admin中管理数据源
- 只启用需要的数据源
- 定期检查数据源状态

### 6.3 性能优化

- 如果数据量大，考虑分批采集
- 使用异步任务（Celery）避免阻塞
- 监控任务执行时间，优化爬虫效率

### 6.4 错误处理

- 设置任务重试机制
- 记录失败原因
- 发送告警通知（可选）

## 七、常见问题

### Q1: 定时任务没有执行？

**检查：**
1. Django服务器是否运行
2. 定时任务是否启用（`ENABLE_SCHEDULER = True`）
3. 查看日志文件是否有错误
4. 检查数据库表 `django_apscheduler_djangojob` 是否有任务记录

### Q2: 任务执行失败？

**检查：**
1. 数据源配置是否正确
2. 网络连接是否正常
3. 爬虫代码是否有更新
4. 查看ETL任务记录了解失败原因

### Q3: 如何修改采集时间？

**APScheduler方案：**
编辑 `apps/data_etl/scheduler.py`，修改 `CronTrigger` 参数

**Cron方案：**
编辑crontab，修改时间表达式

### Q4: 如何临时停止定时任务？

**APScheduler方案：**
在Django Admin中禁用任务，或设置 `ENABLE_SCHEDULER = False`

**Cron方案：**
注释掉crontab中的任务行
