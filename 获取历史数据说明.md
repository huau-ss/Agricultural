# 获取最近三个月历史数据说明

## 问题说明

如果发现爬取的数据日期只到某个日期（如2025.12.7），没有获取到最近三个月的数据，可能是以下原因：

1. **API分页限制**：API可能只返回最新的数据，历史数据需要更多分页
2. **日期过滤逻辑**：爬虫可能在遇到旧数据时提前停止
3. **数据去重**：可能重复数据被过滤

## 已优化的功能

### 1. 增强的分页逻辑

- ✅ 增加最大页数到500页
- ✅ 改进日期过滤逻辑，不会因为遇到旧数据就停止
- ✅ 连续检查多页，确保获取完整数据

### 2. 改进的日期解析

- ✅ 支持多种日期格式
- ✅ 正确处理带时间的日期字符串
- ✅ 更准确的日期范围判断

### 3. 数据去重

- ✅ 根据产品名称、市场名称、日期去重
- ✅ 避免重复数据

## 使用方法

### 方法一：运行爬虫命令

```bash

python manage.py run_spider --source-id 1 --task-name "获取最近三个月数据"

# 或使用定时采集命令
```

## 如果数据仍然不完整

### 1. 检查日志

查看 `logs/agricultural.log`，查找：
- 数据日期范围信息
- 分页信息
- 错误信息

### 2. 手动调整日期范围

如果需要获取更长时间的数据，可以修改 `apps/data_etl/spiders/pfsc_spider.py`：

```python
# 修改第99行，将90天改为更多天
three_months_ago = today - timedelta(days=180)  # 改为6个月
```

### 3. 增加最大页数

如果数据量很大，可以增加最大页数：

```python
# 修改第110行
max_pages = 1000  # 改为更大的值
```

### 4. 检查API限制

某些API可能有：
- 每日请求次数限制
- 单次请求数据量限制
- 需要认证或token

## 常见问题

### Q1: 为什么只获取到某一天的数据？

**A:** 可能是：
1. API返回的数据是按日期分组的，需要更多分页
2. 日期解析失败，所有数据被当作当天数据
3. 数据源网站只提供最近几天的数据

**解决：**
- 检查日志中的日期范围信息
- 手动验证API返回的数据日期
- 增加分页数量

### Q2: 如何获取更长时间的历史数据？

**A:** 修改爬虫代码中的日期范围：

```python
# 在 pfsc_spider.py 中
three_months_ago = today - timedelta(days=180)  # 改为6个月或更长
```

### Q3: 数据获取很慢怎么办？

**A:** 
- 减少请求延迟：修改 `self.sleep(0.3)` 为更小的值
- 增加每页数据量：修改 `page_size = 100` 为更大的值（如果API支持）
- 使用多线程（需要额外开发）

### Q4: 如何验证数据是否完整？

**A:** 使用上面的验证脚本，检查：
1. 数据日期范围是否覆盖三个月
2. 每天的数据量是否合理
3. 是否有日期缺失
