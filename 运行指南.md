## 三、后端部署步骤

### 3.1 创建虚拟环境

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Linux/macOS:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3.2 安装Python依赖

```bash
# 进入项目根目录
cd Agricultural

# 安装依赖
pip install -r requirements.txt
```

**注意**: 如果安装`mysqlclient`失败，可以：
- Windows: 下载对应的wheel文件或使用`pymysql`替代
- Linux: `sudo apt-get install python3-dev default-libmysqlclient-dev`
- macOS: `brew install mysql-client`

### 3.3 配置数据库

1. **创建MySQL数据库**
```sql
CREATE DATABASE agricultural_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. **修改数据库配置**

编辑 `agricultural_platform/settings.py` 中的数据库配置：

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'agricultural_db',
        'USER': 'root',          # 修改为你的MySQL用户名
        'PASSWORD': 'your_password',  # 修改为你的MySQL密码
        'HOST': 'localhost',
        'PORT': '3306',
        'OPTIONS': {
            'charset': 'utf8mb4',
            'init_command': "SET sql_mode='STRICT_TRANS_TABLES'",
        },
    }
}
```

**或者使用pymysql（如果mysqlclient安装失败）:**

在 `agricultural_platform/__init__.py` 中添加：
```python
import pymysql
pymysql.install_as_MySQLdb()
```

### 3.4 执行数据库迁移

```bash
# 生成迁移文件
python manage.py makemigrations

# 执行迁移
python manage.py migrate
```

### 3.5 创建超级用户

```bash
python manage.py createsuperuser
```

按提示输入用户名、邮箱和密码。

### 3.6 创建必要的目录

```bash
# Windows
mkdir logs
mkdir static
mkdir media

# Linux/macOS
mkdir -p logs static media
```

### 3.7 运行开发服务器

```bash
python manage.py runserver
```

服务器将在 `http://127.0.0.1:8000` 启动。

### 3.8 访问Django Admin后台

打开浏览器访问：`http://127.0.0.1:8000/admin/`

使用刚才创建的超级用户登录。

## 四、前端部署步骤

### 4.1 进入前端目录


## 七、常见问题解决

### 7.1 MySQL连接失败

**问题**: `django.db.utils.OperationalError: (2003, "Can't connect to MySQL server")`

**解决**:
- 检查MySQL服务是否启动
- 检查数据库配置是否正确
- 检查防火墙设置

### 7.2 依赖安装失败

**问题**: `mysqlclient` 或 `tensorflow` 安装失败

**解决**:
- 使用国内镜像源：
  ```bash
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- 对于tensorflow，可以降低版本或使用CPU版本

### 7.3 前端跨域问题

**问题**: CORS错误

**解决**: 确保 `settings.py` 中已配置：
```python
CORS_ALLOW_ALL_ORIGINS = True
CORS_ALLOW_CREDENTIALS = True
```

### 7.4 静态文件404

**问题**: 静态文件无法加载

**解决**:
```bash
python manage.py collectstatic
```

### 7.5 数据库迁移错误

**问题**: `django.db.utils.ProgrammingError`

**解决**:
```bash
# 删除迁移文件（谨慎操作）
# 重新生成迁移
python manage.py makemigrations
python manage.py migrate
```
